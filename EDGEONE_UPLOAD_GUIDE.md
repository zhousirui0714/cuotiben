## 🚀 部署方式说明

**EdgeOne Pages 不支持直接上传文件**，而是通过以下方式部署：

### 方式1: 通过 Git 仓库部署（推荐）✅
你的项目已经连接了 Git 仓库，这是 EdgeOne Pages 的标准部署方式。

### 方式2: 通过构建部署
如果你需要上传单个文件，需要通过 Git 提交后自动部署。

## 📋 具体操作步骤

### 方法1: 使用已连接的 Git 仓库（你已经配置好了）

根据截图，你的项目已经连接了 Git 仓库（显示"main"分支）。

1. **确保文件已推送到 GitHub**
   ```bash
   # 在项目目录中执行
   git add .
   git commit -m "Update website files"
   git push origin main
   ```

2. **在 EdgeOne 控制台重新部署**
   - 在你的截图页面，找到"最近部署"区域
   - 点击"构建部署"（左侧菜单）
   - 或者等待自动部署（如果设置了自动部署）

### 方法2: 手动触发部署

1. **点击左侧菜单"构建部署"**
   - 在你的截图左侧边栏，点击"构建部署"
   - 选择"重新部署"或"新建部署"

2. **选择部署方式**
   - 选择 Git 仓库部署
   - 选择分支: main
   - 点击"开始部署"

### 方法3: 配置自动部署

1. **进入项目设置**
   - 点击左侧菜单"项目设置"
   - 找到"自动部署"选项
   - 启用"Git 提交时自动部署"

## 📁 文件位置说明

EdgeOne Pages 的文件不是通过"上传"方式添加的，而是通过：

1. **Git 仓库同步**
   - 文件保存在 GitHub 仓库中
   - EdgeOne 自动从 GitHub 拉取文件
   - 部署后文件存储在 EdgeOne 的 CDN 中

2. **查看已部署的文件**
   - 在 EdgeOne 控制台，点击"构建部署"
   - 查看部署日志和文件列表
   - 或者访问部署后的网站 URL

## 🔍 如何查看已部署的文件

1. **在控制台查看**
   - 点击"构建部署" → 选择最近的部署记录
   - 查看"文件列表"或"构建产物"

2. **通过 URL 访问**
   - 部署完成后，EdgeOne 会提供一个访问链接
   - 格式类似: `https://cuotiben-xxxx.edgeone.app`
   - 在浏览器中直接访问

## ⚡ 快速操作

### 立即部署最新代码

1. 打开你的 EdgeOne 控制台页面
2. 点击左侧菜单 **"构建部署"**
3. 点击 **"重新部署"** 按钮
4. 等待部署完成
5. 点击 **"预览"** 查看网站

### 验证部署成功

- 部署状态显示"成功"
- 可以访问网站 URL
- 页面显示正常

## 💡 提示

- EdgeOne Pages 是**无服务器**的静态网站托管
- 不支持 FTP 或文件管理器上传
- 所有文件必须通过 Git 仓库管理
- 每次推送代码后，EdgeOne 会自动重新部署

## 📞 如果仍有问题

1. 检查 GitHub 仓库是否已更新
2. 在 EdgeOne 控制台查看部署日志
3. 确认分支名称是否正确（main 或 master）
4. 检查构建设置是否正确
